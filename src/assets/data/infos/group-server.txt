내용 3줄 요약
-----------
1. 주 서비스를 사용하려면 이 서버가 연결되어 있어야 함
2. 오픈소스 서버로 누구든 서버를 만들어 운영할 수 있음
3. ID는 이메일, 비밀번호는 기기 고유 ID이며 기기 변경시 이메일 주소로 기기 변경 절차 진행
-----------

 회원정보를 관리하는 Nakama 서버에 연결합니다. 이 앱에서 사용할 수 있는 모든 회원제 서비스는 이 서버에 로그인한 정보를 토대로 행동합니다

 Nakama 서버는 오픈소스 서버로, 원한다면 누구든지 사설서버를 생성하고 관리할 수 있습니다.
 이 앱에서는 사용자의 개인정보를 최소한으로 수집하기 위해 ID로 이메일을, 비밀번호로 기기 고유 ID를 사용합니다.

  이 분야에 관심이 없으신 분들이라면 기기 고유 ID라는 이름을 듣고 있으니 뭔가 이 앱을 실행하는 기기의 매우 소중한, 고귀한 ID라고 생각이 들 수도 있으나 기기 고유 ID라는 것은 기기 정보(하드웨어 정보, 사용자 계정 ID 등)를 활용하여 만든 난잡한 문자열로 OS에서 기본으로 제공해주는 기능입니다.
 보통은 모바일 게임에서 볼 수 있는 "게스트로 입장하기" 같은 기능을 구현할 때 많이 사용합니다.

 안드로이드 기기 고유 ID의 모양은 아래와 비슷하게 생겼습니다.

    예시) a9d9c78fbf8713da180e2ca1

 위와 같은 정보를 통해서 실제 기기 정보를 알 수 있을리가 없지요..

 이메일 주소는 이메일 형식에만 맞추면 검증없이 바로 회원가입으로 처리합니다.
 주 사용 기기가 변경되었다면 등록된 이메일로 기기 변경 등록 메일을 발송해드립니다. 절차를 진행하여 다음 기기를 주 사용 기기로 사용할 수 있습니다.

 개발 테스트 서버는 이름 그대로 개발 테스트 목적이 강하며 개발자 개인 컴퓨터를 사용하고 있어, 개발자 판단에 의해 언제든 운영중지되거나 데이터 초기화가 이루어질 수 있습니다. 이 앱에서 사용하는 서비스로 개발 테스트 서버를 사용하는 것은 서버에 저장된 개인정보 및 활동정보가 고지없이 손실될 수 있다는 것에 동의한 것으로 간주됩니다.

 서버는 Nakama 서버를 이용하고 있으며 통신 보안처리가 이루어져있지 않습니다. (http 사용)

* 사실상 열린 개인정보 고지
 그룹 서버는 Nakama 오픈소스 서버를 이용중으로 누구나 쉽게 서버를 구축할 수 있으며, 해당 서버 관리자라면 Nakama Console에서 지원하는 사용자 프로필 정보, 그룹 정보, 사용자 저장데이터, 리더보드, 매칭 정보에 쉽게 접근할 수 있습니다. 그러니 이 앱을 사용할 때 중요 개인정보가 노출되지 않도록 각별히 신경써주세요.
 
 또한 이 앱은 사용자의 Nakama 사설서버 구축을 장려하고자 Nakama 서버에서 제공하는 기본 기능을 최대한 활용하는 방향으로 앱을 구성중입니다. 그러나 사설 서버 관리자가 Nakama 서버에 추가 기능을 구성한다면 서버와 통신하는 내용을 검토/변조/검색할 수 있음을 숙지하셔야 합니다.
 
 이러한 구조상의 장단점이 있으므로 사용자분께서는 신뢰할 수 있는 기관, 회사, 집단, 그룹 등에서 제공하는 서버를 이용할 것을 강력히 권고드립니다.
